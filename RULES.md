# Development Rules

## Rule 1

Never skip tasks.

Always complete the first incomplete task.

---

## Rule 2

Never implement future features.

Only work on Phase 1.

---

## Rule 3

Always run the application after implementation.

Fix all runtime errors.

---

## Rule 4

Run tests.

Fix failing tests.

---

## Rule 5

Use clean architecture.

---

## Rule 6

No hardcoded values.

Use configuration files.

---

## Rule 7

Write reusable code.

---

## Rule 8

Follow SOLID principles.

---

## Rule 9

Write proper logging.

---

## Rule 10

Handle errors gracefully.

---

## Rule 11

Before marking a task complete verify:

- Builds successfully
- Runs successfully
- No lint issues
- Acceptance criteria satisfied

---

## Rule 12

If blocked

Explain why.

Do not invent fake implementations.

---

## Rule 13

Update TASKS.md after every task.

---

## Rule 14

Never rewrite working code unless necessary.

---

## Rule 15

Always think before coding.

Understand the existing implementation first.
